# SSH & 文件传输 — 功能规约

> 分支：2-ssh-file-transfer
> 创建日期：2026-03-25
> 状态：草稿

## 概述

SSH & 文件传输是 TerminalMind 的第二个里程碑。目标是在 Phase 1 已建立的多标签本地终端和扩展架构基础上，新增 SSH 远程连接、SFTP 文件传输和连接配置管理三大核心能力。这三项能力均以内置扩展（ext-ssh、ext-sftp、ext-connections）形式实现，使用与 Phase 1 ext-terminal 完全相同的 Extension API 注册路径，进一步验证"插件平等可扩展"原则。本阶段的核心价值在于将 TerminalMind 从单纯的本地终端工具升级为可管理远程服务器的一站式终端平台。

## 用户场景

### 参与者

| 参与者 | 描述 |
|---|---|
| 全栈开发者 | 需要频繁连接远程服务器进行开发、部署和运维的主要用户 |
| 运维工程师 | 管理多台服务器，需要批量连接和文件传输的高级用户 |

### 用户故事

#### US1：SSH 远程连接（P1）

**作为**全栈开发者，**我想要**通过 SSH 连接到远程服务器并在终端中进行交互，**以便**在统一的工具中管理本地和远程终端而无需切换到 PuTTY 或其他 SSH 客户端。

**验收场景：**

1. **场景**：使用密码连接远程服务器
   - **假设** 用户已在连接管理器中配置了一台使用密码认证的远程服务器
   - **当** 用户双击该连接或通过命令面板执行 `ssh.connect`
   - **则** 应用建立 SSH 连接，在新标签页中打开远程终端会话，用户可像操作本地终端一样输入命令

2. **场景**：使用密钥连接远程服务器
   - **假设** 用户配置了使用 SSH 私钥认证的连接
   - **当** 用户发起连接
   - **则** 应用使用指定的私钥文件进行认证，如果私钥有密码保护则弹出输入框，连接成功后打开远程终端

3. **场景**：使用 SSH Agent 连接
   - **假设** 用户系统中已运行 SSH Agent 并加载了密钥
   - **当** 用户以 Agent 认证方式发起连接
   - **则** 应用通过 SSH Agent 完成认证，无需手动输入密码或指定密钥路径

4. **场景**：多跳跳板机连接
   - **假设** 用户需要通过跳板机访问内网服务器，已配置跳板机链路
   - **当** 用户发起连接
   - **则** 应用依次通过跳板机建立隧道，最终连接到目标服务器，过程对用户透明

5. **场景**：连接断开处理
   - **假设** SSH 连接因网络中断而断开
   - **当** 连接中断被检测到
   - **则** 终端标签显示"已断开"状态，提供"重新连接"按钮，用户可一键恢复连接

#### US2：端口转发（P2）

**作为**全栈开发者，**我想要**在 SSH 连接上设置端口转发，**以便**访问远程服务器上运行的 Web 服务或数据库而无需暴露端口到公网。

**验收场景：**

1. **场景**：创建本地端口转发
   - **假设** 用户已有一个活跃的 SSH 连接
   - **当** 用户通过命令面板或 UI 指定本地端口和远程端口
   - **则** 端口转发建立成功，用户可通过 `localhost:<本地端口>` 访问远程服务

2. **场景**：查看和管理端口转发
   - **假设** 用户已建立一个或多个端口转发
   - **当** 用户在侧边栏查看连接详情
   - **则** 显示所有活跃的端口转发列表，每条可单独关闭

#### US3：SFTP 文件传输（P1）

**作为**全栈开发者，**我想要**通过 SFTP 在本地和远程服务器之间浏览和传输文件，**以便**在一个工具中完成文件管理而无需另开 FileZilla 或 WinSCP。

**验收场景：**

1. **场景**：浏览远程文件系统
   - **假设** 用户已建立 SSH 连接
   - **当** 用户在侧边栏打开该连接的 SFTP 视图
   - **则** 显示远程服务器的文件目录树，支持展开文件夹、查看文件大小和修改时间

2. **场景**：上传文件到远程服务器
   - **假设** 用户在 SFTP 文件浏览器中选中了目标远程目录
   - **当** 用户通过拖拽或点击按钮选择本地文件上传
   - **则** 文件上传开始，显示进度条和传输速率，完成后远程目录自动刷新

3. **场景**：从远程服务器下载文件
   - **假设** 用户在远程文件列表中选中了一个或多个文件
   - **当** 用户点击下载或拖拽到本地面板
   - **则** 文件下载到指定本地目录，显示进度和速率，完成后提供打开文件位置的入口

4. **场景**：远程文件操作
   - **假设** 用户在远程文件列表中右键点击文件/文件夹
   - **当** 用户选择重命名、删除或新建文件夹
   - **则** 操作在远程服务器上执行，UI 即时更新反映变更

5. **场景**：大文件传输中断恢复
   - **假设** 大文件传输过程中网络短暂中断
   - **当** 网络恢复后
   - **则** 传输队列中的任务显示失败状态，用户可选择重试，重试从头开始传输

#### US4：连接管理器（P1）

**作为**运维工程师，**我想要**一个集中管理所有远程连接配置的面板，支持分组和搜索，**以便**在管理大量服务器时快速找到并连接目标机器。

**验收场景：**

1. **场景**：创建新连接
   - **假设** 用户在连接管理器中点击"新建连接"
   - **当** 用户填写主机、端口、用户名、认证方式等信息
   - **则** 连接配置保存成功，出现在侧边栏连接树中

2. **场景**：编辑已有连接
   - **假设** 用户右键点击连接树中的一个连接
   - **当** 用户选择"编辑"并修改配置
   - **则** 配置更新保存，下次连接使用新配置

3. **场景**：连接分组管理
   - **假设** 用户有大量连接配置
   - **当** 用户创建分组并将连接拖拽或分配到分组中
   - **则** 连接按分组在树视图中组织展示，支持折叠/展开

4. **场景**：搜索连接
   - **假设** 用户在连接列表顶部的搜索框中输入关键词
   - **当** 输入匹配的主机名、连接名称或标签
   - **则** 树视图实时过滤，仅显示匹配的连接

5. **场景**：导入/导出连接配置
   - **假设** 用户需要在多台电脑间同步连接配置
   - **当** 用户导出为 JSON 文件并在另一台电脑导入
   - **则** 所有连接配置（不含明文密码）完整迁移

6. **场景**：快速连接
   - **假设** 用户需要临时连接一台新服务器
   - **当** 用户通过命令面板执行 `ssh.quick-connect` 并输入 `user@host`
   - **则** 无需保存配置即可直接发起 SSH 连接

#### US5：连接状态监控（P2）

**作为**运维工程师，**我想要**实时看到每个连接的状态（已连接/已断开/连接中），**以便**快速了解当前服务器连接情况。

**验收场景：**

1. **场景**：连接状态实时更新
   - **假设** 侧边栏连接树已显示多个连接
   - **当** 某个连接状态从"已断开"变为"已连接"或反过来
   - **则** 连接图标即时更新反映当前状态，无需手动刷新

2. **场景**：状态栏连接信息
   - **假设** 用户当前活跃标签是一个 SSH 远程终端
   - **当** 用户查看状态栏
   - **则** 状态栏显示当前连接的主机名、用户名和连接时长

| 标准 | 指标 | 目标值 |

## 范围边界

### 包含

- SSHService 完整实现（密码/密钥/Agent 认证、多跳、端口转发、保活）
- SFTPService 实现（文件浏览、上传、下载、删除、重命名、创建目录）
- ConnectionStore 实现（CRUD、分组、JSON 导入导出）
- ext-ssh 内置扩展（ssh.connect、ssh.disconnect、ssh.quick-connect、ssh.forward-port）
- ext-sftp 内置扩展（sftp.open、sftp.upload、sftp.download）
- ext-connections 内置扩展（connections.create、connections.edit、connections.delete、connections.import、connections.export）
- 侧边栏连接树视图（分组、状态图标、搜索）
- SFTP 文件浏览器面板（双面板布局：本地+远程）
- 敏感信息安全存储（系统 Keychain 集成）
- IPC 通道扩展（SSH 和 SFTP 相关通道）
- Extension API 扩充（connections 命名空间、terminal 命名空间增强）
- 新增 SSH/SFTP/Connection 相关单元测试

### 排除

- AI 命令生成和对话功能（Phase 3）
- PipelineEngine 管道引擎完整实现（Phase 3）
- 插件市场和第三方插件加载（Phase 4）
- Extension Worker 隔离机制（Phase 4）
- PermissionManager 权限管理（Phase 4）
- MobaXterm / PuTTY 配置导入（Phase 5，本阶段仅 JSON 格式）
- SFTP 断点续传（技术复杂度高，首个版本不实现）
- 远程文件编辑（Phase 5 计划中的下载→外部编辑器→回传功能）
- SSH 远程端口转发（Remote Forward）和动态转发（SOCKS Proxy）
- 终端分屏功能
- 批量 SSH 命令执行（Phase 3 Pipeline 功能）

## 边界情况

- **SSH 连接超时**：网络不可达或服务器未开放 SSH 端口时，应在合理时间（默认 10 秒）内超时并显示明确的错误信息，不无限等待
- **认证失败**：密码错误、密钥不匹配或权限被拒时，显示具体的认证失败原因，提供重试选项，密码输入不超过 3 次
- **主机密钥验证**：首次连接未知主机时，显示主机指纹并请求用户确认，已知主机指纹变更时显示安全警告
- **连接配置文件损坏**：connections.json 格式损坏时，应用应能启动并告知用户，提供备份恢复选项
- **Keychain 不可用**：当系统 Keychain 服务不可用时（如 Linux 无 libsecret），降级为加密文件存储或提示用户安装依赖
- **SFTP 传输中断**：大文件传输过程中 SSH 连接断开时，清理临时文件，传输任务标记为失败
- **远程文件权限不足**：尝试操作无权限的远程文件/目录时，显示清晰的权限拒绝信息
- **极长路径名**：远程文件系统中出现超长路径或特殊字符文件名时，UI 正确截断显示，操作不崩溃
- **并发 SFTP 操作**：多个传输任务同时进行时，共享同一 SFTP 通道或正确管理通道池，避免互相干扰
- **跳板机链路中断**：多跳连接中某一中间节点断开时，整个链路正确清理，不留下僵尸连接

## 依赖

- **ssh2**（Node.js）：纯 JavaScript 实现的 SSH2 客户端，用于 SSH 连接和 SFTP
- **keytar**：跨平台系统 Keychain 访问库（macOS Keychain / Windows Credential Manager / Linux libsecret）
- Phase 1 已有基础设施（Core CLI、Extension API、TerminalService、GUI Shell 布局、IPC Bridge）
- Phase 1 已有 ext-terminal 作为 Extension API 使用范例

## 假设

- 目标远程服务器运行标准 OpenSSH 兼容的 SSH 服务端（版本 5.3+）
- 用户网络环境允许出站 TCP 连接到远程服务器的 SSH 端口（默认 22）
- Phase 2 仅实现 JSON 格式的连接导入导出，MobaXterm/PuTTY 格式推迟到 Phase 5
- SFTP 传输不实现断点续传，失败后用户需重新传输整个文件
- 密钥文件支持 OpenSSH 格式（id_rsa、id_ed25519 等），PuTTY PPK 格式暂不支持
- 主机密钥存储在 ~/.terminalmind/known_hosts 文件中，格式参考 OpenSSH known_hosts
- 端口转发仅实现本地端口转发（-L 模式），远程端口转发和动态转发推迟到后续阶段
