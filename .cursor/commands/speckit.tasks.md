**语言规则**: 生成的 tasks.md 中所有自然语言内容必须使用简体中文编写。包括阶段标题、任务描述、目标、检查点、依赖说明、策略描述和注意事项。技术标识符（任务 ID 如 T001、标记如 [P]/[US1]、文件路径、命令示例）保持英文。

2. **加载设计文档**：从 FEATURE_DIR 读取：
   - **必需**：plan.md（技术栈、库、结构）、spec.md（带优先级的用户故事）
   - **可选**：data-model.md（实体）、contracts/（接口契约）、research.md（决策）、quickstart.md（测试场景）
   - 注意：并非所有项目都有全部文档。根据可用内容生成任务。

3. **执行任务生成工作流**：
   - 加载 plan.md，提取技术栈、库、项目结构
   - 加载 spec.md，提取用户故事及其优先级（P1、P2、P3 等）

4. **生成 tasks.md**：使用 `.specify/templates/tasks-template.md` 作为结构，填入：

   - 每个阶段包含：故事目标、独立测试标准、测试（如有要求）、实现任务

## 任务生成规则

**关键**：任务必须按用户故事组织，以支持独立实现和测试。

**测试为可选项**：仅在功能规约中明确要求或用户要求 TDD 方式时才生成测试任务。

### 检查清单格式（必需）

每个任务必须严格遵循以下格式：

```text
- [ ] [TaskID] [P?] [Story?] 描述（含文件路径）
```

**格式组成**：

1. **复选框**：始终以 `- [ ]` 开头（Markdown 复选框）
2. **任务 ID**：按执行顺序的序号（T001、T002、T003...）
3. **[P] 标记**：仅在任务可并行时包含（不同文件，不依赖未完成的任务）
4. **[Story] 标签**：仅在用户故事阶段的任务中必需
   - 格式：[US1]、[US2]、[US3] 等（映射到 spec.md 中的用户故事）
   - 初始化阶段：无故事标签
   - 基础层阶段：无故事标签
   - 用户故事阶段：必须有故事标签
   - 收尾阶段：无故事标签
5. **描述**：明确的操作，包含确切文件路径

**示例**：

- ✅ 正确：`- [ ] T001 按实现计划创建项目结构`
- ✅ 正确：`- [ ] T005 [P] 实现认证中间件 in src/middleware/auth.py`
- ✅ 正确：`- [ ] T012 [P] [US1] 创建 User 模型 in src/models/user.py`
- ✅ 正确：`- [ ] T014 [US1] 实现 UserService in src/services/user_service.py`
- ❌ 错误：`- [ ] 创建 User 模型`（缺少 ID 和故事标签）
- ❌ 错误：`T001 [US1] 创建模型`（缺少复选框）
- ❌ 错误：`- [ ] [US1] 创建 User 模型`（缺少任务 ID）
- ❌ 错误：`- [ ] T001 [US1] 创建模型`（缺少文件路径）

### 任务组织

1. **来自用户故事（spec.md）**——主要组织方式：
   - 每个用户故事（P1、P2、P3...）获得自己的阶段
   - 将所有相关组件映射到其故事：
     - 该故事需要的模型
     - 该故事需要的服务
     - 该故事需要的接口/UI
     - 如有测试要求：该故事特定的测试
   - 标记故事依赖（大多数故事应独立）

2. **来自契约**：
   - 将每个接口契约映射到它服务的用户故事
   - 如有测试要求：每个接口契约 → 该故事阶段中实现前的契约测试任务 [P]

3. **来自数据模型**：
   - 将每个实体映射到需要它的用户故事
   - 如果实体服务多个故事：放在最早的故事或初始化阶段
   - 关系 → 相应故事阶段的服务层任务

4. **来自初始化/基础设施**：
   - 共享基础设施 → 初始化阶段（Phase 1）
   - 基础/阻塞任务 → 基础层阶段（Phase 2）
   - 故事特定的初始化 → 在该故事的阶段内

### 阶段结构

- **Phase 1**：初始化（项目初始化）
- **Phase 2**：基础层（阻塞前置条件——必须在用户故事前完成）
- **Phase 3+**：按优先级排序的用户故事（P1、P2、P3...）
  - 每个故事内部：测试（如有要求）→ 模型 → 服务 → 端点 → 集成
  - 每个阶段应是完整的、可独立测试的增量
- **最终阶段**：收尾与横切关注点
